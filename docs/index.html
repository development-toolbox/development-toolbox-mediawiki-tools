üìö How-To Guides

Step-by-step guides for common migration scenarios

üéØ Choose Your Scenario: Select the guide that best matches your situation.

üöÄ Quick Migration (10-50 pages)

Best for: Small team wikis, simple documentation

Step-by-Step Commands
# 1. Quick setup
python getting_started.py

# 2. Analyze your wiki
python migration/migration_planner.py --wiki "TeamDocs"

# 3. Preview key pages
python migration/content_previewer.py --page "Home"

# 4. Run migration
python migration/azure_devops_migrator.py --wiki "TeamDocs"

Expected time: 15-30 minutes

Beginner Friendly

üè¢ Department Migration (100-500 pages)

Best for: Large teams, complex content structure

Phased Approach
# 1. Comprehensive analysis
python migration/migration_planner.py --wiki "DeptWiki" --detailed

# 2. Preview multiple pages
python migration/content_previewer.py --sample-pages 20

# 3. Test with small batch
python migration/azure_devops_migrator.py --wiki "DeptWiki" \
    --batch-size 10 --dry-run

# 4. Migrate in phases
python migration/azure_devops_migrator.py --wiki "DeptWiki" \
    --batch-size 25 --exclude-pattern "Archive/*"

# 5. Validate results
python migration/validation_tool.py --wiki "DeptWiki"

Expected time: 2-4 hours

Intermediate

üåê Enterprise Migration (500+ pages)

Best for: Organization-wide wikis, legacy content

Strategic Migration
# 1. Strategic analysis
python migration/migration_planner.py --wiki "EnterpriseWiki" \
    --detailed --export-analysis

# 2. Current content first
python migration/azure_devops_migrator.py --wiki "EnterpriseWiki" \
    --modified-after "2023-01-01" \
    --exclude-pattern "Archive/*,Deprecated/*" \
    --batch-size 10

# 3. Archive content separately
python migration/azure_devops_migrator.py --wiki "EnterpriseWiki" \
    --include-pattern "Archive/*" \
    --target-namespace "Archive" \
    --batch-size 5

Expected time: 1-2 days

üìä Content Analysis Deep Dive

üéØ Selective Migration

Targeted Content Migration
# Migrate only recent content
python migration/azure_devops_migrator.py --wiki "MyWiki" \
    --modified-after "2024-01-01"

# Migrate specific pages only
python migration/azure_devops_migrator.py --wiki "MyWiki" \
    --pages "Home,Setup,FAQ,Troubleshooting"

# Exclude patterns
python migration/azure_devops_migrator.py --wiki "MyWiki" \
    --exclude-pattern "Draft/*,Archive/*,Temp/*"

üîÑ Resume and Recovery

Handle Interruptions
# Resume from specific page

    --resume-from "Page Name"

# Process failed pages only
python migration/azure_devops_migrator.py --wiki "MyWiki" \
    --retry-failed

# Custom retry strategy
python migration/azure_devops_migrator.py --wiki "MyWiki" \
    --max-retries 5 --retry-delay 10

‚ö° Performance Optimization

Speed Optimization

    --batch-size 50 --rate-limit 0.5

# Conservative migration (large wikis)

üõ†Ô∏è Troubleshooting

Common issues and solutions

üö® Authentication Issues

Problem: Unable to connect to Azure DevOps or MediaWiki

Solutions:

  • Verify Personal Access Token has proper permissions
  • Check MediaWiki API credentials
  • Ensure network connectivity
  • Validate .env file configuration

üìÑ Content Conversion Errors

Problem: Content not converting properly

Solutions:

  • Use content previewer to identify issues
  • Check for unsupported HTML elements
  • Validate markdown syntax in source
  • Review complex table structures

üñºÔ∏è Image Migration Issues

Problem: Images not displaying correctly

Solutions:

  • Images require manual download/upload
  • Update image references after upload
  • Check MediaWiki file upload permissions
  • Verify image file formats are supported

‚ö° Performance Issues

Problem: Migration taking too long

Solutions:

  • Use batch processing for large wikis
  • Check API rate limits
  • Optimize network connectivity
  • Consider running during off-peak hours
üí° Pro Tip: Always test with a small batch of pages first, and keep backups of your original content before migration.
